java学习那些事儿(1)

记录下用java刷题过程中遇到的一些细的散的知识点。

1.java运算符中的与(&)、或(|)、异或(^)

与(&)、或(|)、异或(^)三种运算都用于java的二进制运算中。与(&)运算：两个数都转为二进制后从高位到低位依次各个位比较，都为1则赋1，否则赋0。e.g. 9&3:按八位来转为二进制分别为00001001和00000011,按照运算规则计算得00000001，再转化为十进制即为1；或(|)运算：两个数都转为二进制后从高位到低位依次各个位比较，两位中只要有1就赋1，否则赋0。e.g. 9|3:按八位来转为二进制分别为00001001和00000011,按照运算规则计算得00001011，再转化为十进制即为11；异或(^)运算：两个数都转为二进制后从高位到低位依次各个位比较，两位中只要不同就赋1，相同则赋0。e.g. 9^3:按照运算规则计算得00001010，再转化为十进制即为10。

2.List中的get()用法

    List<Integer> list = new ArrayList<>();
    list.add(1); //此处其实存在自动包装机制，即为list.add(new Integer(1))；
    list.add(2);
    list.add(3);
    for (int i = 0; i < list.size(); i++){
        System.out.println(list.get((i)));
    }

get()方法中，get(i)表示取list中第i个元素，当i=0时，取第一个，i=2取第二个，以此类推，类似数组下标，从0开始，需要注意越界的问题。

3.Queue中的poll()与remove()区别

Queue中poll()与remove()方法都是从Queue头部中删除一个元素并返回，不同的是，在队列元素为空的情况下，remove()会抛出NoSuchElementException异常，而poll()不会抛出异常，只会返回null。

4.Operator ‘&’ cannot be applied to ‘int’, ‘boolean’错误

System.out.println(3 & 1 == 0); //报错

java中关系运算符优先级要大于逻辑运算符，因此会计算1 == 0等于false，再计算3 & false。&左右两边必须类型相同，都是int型或者都是boolean型，int & boolean显然错误。正确的方法是在3 & 1添括号，变成(3 & 1) == 0。

单元数组cell的用法

单元数组可以将多个不同类型的数据和不同维度的数据存储在一起。具体定义用大括号{}，例如下面：

flower_name = {'风信子','桂花','荷花','樱花','洛神花'};

定义了包含五个字符串的单元数组。如果要调用其中某个元素的话，和普通的数组[]不同，需要用flower_name{1}来表示第一个元素，如果使用flower_name(1)，表示的是cell属性的值。

strcat

I1 = strcat('a','b','c');

dir

dir()函数可以获得指定文件夹下所有的子文件夹和文件，并存放在一种叫做文件结构体的数组中。例如：

files = dir('D:/new_flowers/','*.jpg'));

上面的代码描述了获取D:/new_flowers中的所有jpg格式的图片，并将其存在文件结构体数组中。

matlab实现图片插值缩放

很久没更新博客了，这段时间没空刷leetcode题，在做实验室的课题，随便帮老师做一些杂事，前几天便涉及到了一个用插值法来做缩放图片的小任务，帮老师解决了下，同时也学到了一些图片缩放的知识，便想着把学到的东西记录下来供日后回顾。

图片缩放

图片缩放，分为图片缩小和图片放大。众所周知，图片是由一个个的像素点构成的，分辨率越高的图，像素点越多。有时根据不同的需求，我们需要将图片放大或者缩小。那么如何实现呢？很简单，对于缩小图片而言，我们少取一些像素点，所构成的图像素点变少，不就把图片变小了嘛；同理，对于放大图片，我们把空缺的地方补上像素点，得到的图就是放大后的图了。因此，采样法与插值法都是专注于如何取像素点与补像素点。我看到网上有博客说图片缩小称为降采样，放大则称为升采样或者插值。我个人觉得这是不准确的，因为插值法可以用来图片放大，也可以用来缩小图片；同样采样法不光能用在图片缩小，也能用在图片放大（虽然我没找到采样法放大图的具体方法）。

降采样缩小图片

网上有降采样的例子，我就直接摘抄代码了，思想其实就是间隔若干行/列来取像素点。以一幅1616的图片为例，有1616=256个像素点，我们可以采用每行隔一个像素点后取值，再每隔一行继续取，即取(1,1),(1,3),…(1,255),(3,1),(3,3)…，将其组合后，就得到8*8=64的图片了，这即是缩小两倍的降采样图。

代码实现：

I=imread('example.jpg');
I1=I(1:2:end,1:2:end);%每2位采集1位
I2=I(1:4:end,1:4:end);%每4位采集1位
I3=I(1:8:end,1:8:end);%每8位采集1位
subplot(221), imshow(I),title('原图');    
subplot(222), imshow(I1),title('缩小2倍');    
subplot(223), imshow(I2),title('缩小4倍');
subplot(224), imshow(I3),title('缩小8倍');

插值法放大/缩小图片

插值法的思想是基于目标(想要放大/缩小的图)分辨率中的像素点，按照缩放关系映射到原图片中，寻找原图中的像素点，然后通过原图中的像素点进行插值，找到目标图中要放入的点位置，将像素点填入。常见的图像插值算法包括最近邻插值（Nearest-neighbor）、双线性插值（Bilinear）、双立方插值（bicubic），这三种插值方法metlab都有集成，我就直接贴代码了。

代码实现：

I=imread('example.jpg');
I1=imresize(I1,2,'nearest');%最近邻插值放大2倍
I2=imresize(I1,2,'bilinear');%双线性插值放大2倍
I3=imresize(I1,2,'bicubic');%双立方插值放大2倍
subplot(221), imshow(I),title('原图');    
subplot(222), imshow(I1),title('最近邻插值');    
subplot(223), imshow(I2),title('双线性插值');
subplot(224), imshow(I3),title('双立方插值');

总结：一些疑问

如何比较插值或者采样的好坏呢？我看到一篇文献里比较不同插值缩放图片的步骤里，效果图里，目标图与原图是比例一致，大小相同的，我很好奇这样是怎么做的？

按理说图片放大或者缩小后，由于大小问题，确实不好与原图比较效果，那文章是如何把图片调整到同一比例的呢？那只能是对缩小后的图又用一次放大方法使其与原图相同了，那这样使用到的插值或者采样二次不会对结果造成误差嘛？另外，windows系统里自带的图片放大缩小功能又是使用的什么方法呢？搞不懂。。

LeetCode的那些事儿-java栈1

把LeetCode中栈解法的简单题型都过了一遍，所以想把一些比较经典的题目解法和栈的思想给记录下来。

栈介绍

定义

栈，简单来说就是一个类似数组的容器，把元素都放进栈中，所有元素按照先进后出的顺序排列。以[a, b, c]为例，依次向栈中存入a、b、c，则取出顺序是c、b、a，最后进的最先出，最先进的最后出。

java栈

java中实现了几种栈的用法，以Object类型为例，如下：

1.Stack<Object> stack = new Stack<>();
2.Deque<Object> stack = new ArrayDeque<Object>();
3.Deque<Object> stack = new LinkedList<Object>();

其中，ArrayDeque和LinkedList既可以实现栈，也可以用来实现队列。

栈的几种重要函数

java栈有几种重要函数是在LeetCode中需要经常用到的，如下：

1.stack.isEmpty() //判断栈是否为空
2.stack.push() //入栈，一般返回空类型
3.stack.pop() //出栈，有返回出栈元素，也有返回空类型的
4.stack.peek() //返回栈顶元素
5.stack.size() //返回栈容量

几个经典栈解法的题

1.最小栈

问题描述：设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈。
解：首先，我们可以想到，每次存入一个元素时，栈都能保存当前栈中最小的元素，为了实现这个想法，我们在构造器中定义两个栈，其中一个栈是辅助栈，另一个是元素栈，辅助栈与元素栈同步进出栈。在辅助栈栈顶存入最小元素，每次入栈时进行判断，只要有比辅助栈顶更小的元素就入栈；出栈时，辅助栈的元素也一同出栈。这样每个时刻辅助栈顶都能存放到最小元素，直接调用peek()方法即可。

代码实现：

class MinStack {
    Deque<Integer> xstack;
    Deque<Integer> minstack;
    /** initialize your data structure here. */
    public MinStack() {
        xstack = new LinkedList<Integer>();
        minstack = new LinkedList<Integer>();
        minstack.push(Integer.MAX_VALUE);
    }

    public void push(int x) {
        xstack.push(x);
        minstack.push(Math.min(minstack.peek(),x));
    }

    public void pop() {
        xstack.pop();
        minstack.pop();
    }

    public int top() {
        return xstack.peek();

    }

    public int getMin() {
        return minstack.peek();
    }
}

2.队列实现栈

题目描述：使用队列实现栈的下列操作：

  • push(x) – 元素 x 入栈
  • pop() – 移除栈顶元素
  • top() – 获取栈顶元素
  • empty() – 返回栈是否为空

这里介绍下，队列是与栈类似的容器，唯一的区别在于队列是先进先出的。

解：同样有两种方法，在add或者remove中进行解决，我们依然考虑在add中进行解决，使用两个队列，思想还是把队列的顺序倒过来，每次入队后，把队里元素颠倒后再存入，队列就变成栈了。具体操作为：先把当前元素进队2（队2与栈2都是初始为空的），注意这里和队列实现栈不同的是，要先把当前元素进队列中。然后再把队1的元素依次出队后再进队2中，之后将队1与队2交换，交换后的队1中，刚入的元素就在队的最前面，即队顶了，而队2再次变为空队。之后remove与top、epmpy操作都正常。需要注意的是记录下每次入队后的当前队顶的元素。

代码实现：

class MyStack {
    private Queue<Integer> queue1;
    private Queue<Integer> queue2;
    private int top;
    /** Initialize your data structure here. */
    public MyStack() {
        queue1 = new LinkedList<>();
        queue2 = new LinkedList<>();
    }

    /** Push element x onto stack. */
    public void push(int x) {
        queue2.add(x);
        top = x;
        while(!queue1.isEmpty()){
            queue2.add(queue1.remove());
        }
        Queue<Integer> temp = queue1; //不用初始化，直接赋值
        queue1 = queue2;
        queue2 = temp;
    }

    /** Removes the element on top of the stack and returns that element. */
    public int pop() { //出队列后，top需要更换
        //return queue1.remove();
        queue1.remove();
        int res = top;
        if (!queue1.isEmpty()) //需要加这行,否则peek方法会报空指针的错误
            top = queue1.peek();
        return res;
    }

    /** Get the top element. */
    public int top() {
        return top;
    }

    /** Returns whether the stack is empty. */
    public boolean empty() {
        return queue1.isEmpty();
    }
}

3.栈实现队列

题目描述：请你仅使用两个栈实现先入先出队列。队列应当支持一般队列的支持的所有操作（push、pop、peek、empty）。
解：有两种方法，在push或者pop中进行解决，考虑在push中进行解决的话，我们就要把栈的顺序倒过来，每次入栈后，把栈里元素颠倒后再存入，栈就变成队列了。具体操作为：先把栈1的元素全部出栈，并进到栈2中，此时再把要入栈的当前元素入栈2，再把所有栈2的元素倒入栈1中，这样当前元素就在栈1的最底下，实现了队列的先进先出，后进后出的效果了。之后pop正常出栈即可。其余peek与empty都正常操作。

代码实现：

class MyQueue {
    private Deque<Integer> stack1;
    private Deque<Integer> stack2;
    private int front;
    /** Initialize your data structure here. */
    public MyQueue() {
        stack1 = new LinkedList<Integer>();
        stack2 = new LinkedList<Integer>();
    }

    /** Push element x to the back of queue. */
    public void push(int x) {
        if(stack1.isEmpty())
            front = x;
        while(!stack1.isEmpty())
            stack2.push(stack1.pop());
        stack2.push(x);
        while(!stack2.isEmpty())
            stack1.push(stack2.pop());
    }

    /** Removes the element from in front of queue and returns that element. */
    public int pop() {
        int res = front;
        stack1.pop();
        if (!stack1.isEmpty())
            front = stack1.peek();
        return res;
    }

    /** Get the front element. */
    public int peek() {
        return front;
    }

    /** Returns whether the queue is empty. */
    public boolean empty() {
        return stack1.isEmpty();
    }
}

LeetCode的那些事儿-题1046

同样是通过刷题来了解的堆的使用，用堆来解。

题目介绍

1046. 最后一块石头的重量

难度：简单

有一堆石头，每块石头的重量都是正整数。

每一回合，从中选出两块最重的石头，然后将它们一起粉碎。假设石头的重量分别为x和y，且x <= y。那么粉碎的可能结果如下：

如果x == y，那么两块石头都会被完全粉碎；
如果x != y，那么重量为x的石头将会完全粉碎，而重量为y的石头新重量为y-x。
最后，最多只会剩下一块石头。返回此石头的重量。如果没有石头剩下，就返回0。

示例

输入：[2,7,4,1,8,1]
输出：1
解释：
先选出 7 和 8，得到 1，所以数组转换为 [2,4,1,1,1]，
再选出 2 和 4，得到 2，所以数组转换为 [2,1,1,1]，
接着是 2 和 1，得到 1，所以数组转换为 [1,1,1]，
最后选出 1 和 1，得到 0，最终数组转换为 [1]，这就是最后剩下那块石头的重量。

提示

1.1 <= stones.length <= 30
2.1 <= stones[i] <= 1000

题解

如果对堆的使用有了一些了解的话，那这道题就很简单了，本身题目也不难理解。我就是先把题703和剑指offer40两道堆相关的题刷完后，再刷这道题，结果直接一遍ac（印象里第一次一遍过！）。具体思路就是定义一个大根堆，然后把数组所有元素都放入堆中，这样每次取出堆顶元素都是最大的。设定while循环条件为堆容量大于1就一直进行下去，每次分别取出两个元素，并相减后判读是否为0，不为0就把差值入堆。最后得到要不是空堆，要不就是只有一个元素的堆，输出即可。

代码(java)

class Solution {
    public int lastStoneWeight(int[] stones) {
        PriorityQueue<Integer> queue = new PriorityQueue<>(new Comparator<Integer>(){
            public int compare(Integer o1, Integer o2){
                return o2 - o1;
            }
        });
        for (int e : stones)
        {
            queue.offer(e);
        }
        int a = 0, b = 0;
        while(queue.size() > 1){
            if (!queue.isEmpty())
                a = queue.poll();
            if (!queue.isEmpty())
                b = queue.poll();
            if ((a - b) != 0)
                queue.offer(Math.abs(a-b));
        }
        if (queue.size() == 0)
            return 0;
        else
            return queue.poll();
    }
}

###后记

LeetCode的那些事儿-剑指 Offer 40

这道题是我入坑堆结构的第一题，方法有很多，最简单也是最容易想到的就是暴力法,直接把数组排序后再输出前k个。因为是按照题型来刷的，所以我采用堆来解。

题目介绍

剑指 Offer 40. 最小的k个数

难度：简单

输入整数数组arr，找出其中最小的k个数。例如，输入4、5、1、6、2、7、3、8这8个数字，则最小的4个数字是1、2、3、4。

示例

输入：arr = [3,2,1], k = 2
输出：[1,2] 或者 [2,1]

输入：arr = [0,1,2,1], k = 1
输出：[0]

说明

  • 0 <= k <= arr.length <= 10000
  • 0 <= arr[i] <= 10000

题解

先构造一个大根堆，然后将前k个元素都入堆，入堆后后面的元素进行判断，如果小于堆顶元素的话，就入堆，并把堆顶元素剔除，否则不变。这样得到的大根堆中的k个元素就是最小的k个元素。然后依次赋值给数组元素并返回即可。需要注意的一点是，若k为0，则返回的是空数组。

代码(java)

class Solution {
    public int[] getLeastNumbers(int[] arr, int k) {
        if (k == 0)
            return new int[0];
        PriorityQueue<Integer> queue = new PriorityQueue<>(new Comparator<Integer>() {
            @Override
            public int compare(Integer o1, Integer o2) {
                return o2 - o1;
            }
        });
        for (int i = 0; i < k; i++){
            queue.offer(arr[i]);
        }
        for (int i = k; i < arr.length; i++){
            if (queue.peek() > arr[i]){
                queue.poll();
                queue.offer(arr[i]);
            }
        }
        int[] res = new int[k];
        for(int i = 0; i < k; i++){
            res[i] = queue.poll();
        }
        return res;
    }
}

###后记
这道题中，我第一次了解到了java中构造优先队列有专门的方法PriorityQueue，这是默认小根堆，即最小堆，如果要用大根堆的话，需要添加Comparator对象。

LeetCode的那些事儿-题703

我打算按照LeetCode的各种题型来刷，把简单的题型都过一遍，正好还可以复习下数据结构。这次是堆结构，题目也都是用的堆来解的。

题目介绍

703. 数据流中的第K大元素

难度：简单

设计一个找到数据流中第K大元素的类（class）。注意是排序后的第K大元素，不是第K个不同的元素。

你的KthLargest类需要一个同时接收整数k和整数数组nums的构造器，它包含数据流中的初始元素。每次调用KthLargest.add，返回当前数据流中第K大的元素。

示例

int k = 3;
int[] arr = [4,5,8,2];
KthLargest kthLargest = new KthLargest(3, arr);
kthLargest.add(3);   // returns 4
kthLargest.add(5);   // returns 5
kthLargest.add(10);  // returns 5
kthLargest.add(9);   // returns 8
kthLargest.add(4);   // returns 8

说明

nums的长度≥k-1且k≥ 1

题解

这道题要求我们写出一个类，类中的方法可以返回排序后的第k个元素，即可以添加新元素，并且每次输出从小到大第k个元素。使用堆来解的话，就需要使用最小堆。我们首先声明一个最小堆和一个整型变量k，在构造器中定义它们，并在构造器中提前将所有元素都使用add方法添加进堆。值得注意的是，构造器中的堆定义里要注明堆的大小为k，因为我们需要在add方法中对入堆元素进行限制：如果选中元素大于堆顶元素（堆顶元素此时是最小元素），则将堆顶元素排出，选中元素入堆。按照这样的规则，小于堆顶的元素不进去，大于的话则剔除对端元素后把选中的入堆，这样的话k个容量的最小堆中，堆顶的元素就一定是从小到大第k个元素。另外还有一个点需要注意的是，可能k会给的很大，导致在构造器中把所有元素入堆后堆容量依旧没满。我们需要在add方法中添加相应的条件。

代码(java)

class KthLargest {
    private PriorityQueue<Integer> queue;
    private int k;
    public KthLargest(int k, int[] nums) {
        this.k = k;
        queue = new PriorityQueue<>(k);
        for(int e : nums) add(e);
    }

    public int add(int val) {
        if (queue.size() < k) 
            queue.offer(val);
        else{
            if (val > queue.peek()){
                queue.poll();
                queue.offer(val);
            }
        }
        return queue.peek();
    }
}
